## 1. Тестирование производительности

### Виды нагрузочного тестирования:

| Вид | Цель | Пример |
|-----|------|--------|
| **Нагрузочное** | Поведение при ожидаемой нагрузке | 1000 одновременных пользователей |
| **Стресс-тестирование** | Предел прочности системы | Увеличиваем до отказа |
| **Тестирование всплесков** | Внезапное увеличение нагрузки | 100 → 5000 → 100 пользователей |
| **Тестирование выносливости** | Стабильность при длительной нагрузке | 500 пользователей, 24 часа |

### Ключевые метрики:

| Метрика | Описание | Хорошо | Плохо |
|---------|----------|--------|-------|
| **Время отклика (P95)** | 95% запросов быстрее | < 500мс | > 3с |
| **Пропускная способность** | Запросов в секунду | Стабильно | Падает под нагрузкой |
| **Частота ошибок** | % ошибок при нагрузке | < 1% | > 5% |
| **CPU/Память** | Использование ресурсов | < 80% | > 95% |

### Пример нагрузочного теста (k6):

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '2m', target: 100 },  // Разгон до 100 пользователей
    { duration: '5m', target: 100 },  // Удержание
    { duration: '2m', target: 0 },    // Снижение
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'],  // 95% запросов < 500мс
    http_req_failed: ['rate<0.01'],    // < 1% ошибок
  },
};

export default function () {
  const res = http.get('https://api.example.com/products');
  check(res, {
    'статус 200': (r) => r.status === 200,
    'время < 500мс': (r) => r.timings.duration < 500,
  });
  sleep(1);
}
```

## 2. Базовые проверки безопасности (OWASP Top 10)

### OWASP Top 10 — что проверять:

| # | Угроза | Что проверить |
|---|--------|--------------|
| 1 | **Нарушение контроля доступа** | Доступ к чужим данным по ID |
| 2 | **Криптографические сбои** | Пароли в открытом виде, HTTP вместо HTTPS |
| 3 | **Инъекции** | SQL-инъекции, XSS в полях ввода |
| 4 | **Небезопасный дизайн** | Отсутствие лимитов, предсказуемые ID |
| 5 | **Ошибки конфигурации** | Стандартные пароли, стектрейсы в ответах |
| 6 | **Уязвимые компоненты** | Устаревшие библиотеки с CVE |
| 7 | **Ошибки аутентификации** | Нет блокировки после перебора |
| 8 | **Нарушение целостности данных** | Небезопасная десериализация |
| 9 | **Недостаточное логирование** | Неудачные входы не логируются |
| 10 | **SSRF** | Сервер делает запросы на внутренние ресурсы |

### Практические проверки:

```
Контроль доступа:
  GET /api/users/1/orders    (мой аккаунт — 200 OK)
  GET /api/users/2/orders    (чужой аккаунт — 403 Forbidden!)

SQL-инъекция:
  Поле email: nastya@test.com' OR '1'='1
  Ожидание: ошибка валидации, НЕ утечка данных

XSS:
  Поле имени: <script>alert('XSS')</script>
  Ожидание: экранирование, скрипт НЕ выполняется

Заголовки безопасности:
  X-Content-Type-Options: nosniff
  X-Frame-Options: DENY
  Content-Security-Policy: ...
  Strict-Transport-Security: max-age=...
```

## 3. Кросс-браузерное и кросс-платформенное тестирование

### Матрица совместимости:

| Браузер | Windows | macOS | Linux | Мобильный |
|---------|---------|-------|-------|-----------|
| **Chrome** | ✅ Основной | ✅ Основной | ✅ | ✅ Android |
| **Firefox** | ✅ | ✅ | ✅ | ✅ |
| **Safari** | ❌ | ✅ Основной | ❌ | ✅ iOS |
| **Edge** | ✅ | ✅ | ✅ | ✅ |

### Типичные кросс-браузерные проблемы:

| Проблема | Пример |
|----------|--------|
| CSS-различия | Flexbox/Grid рендеринг |
| Шрифты | Размеры отличаются между ОС |
| Даты | Формат Date.parse() |
| API браузера | WebRTC, clipboard, notifications |
| Прокрутка | Поведение scroll-snap |

### Автоматизация кросс-браузерного тестирования (Playwright):

```typescript
// playwright.config.ts
export default defineConfig({
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
    { name: 'firefox', use: { ...devices['Desktop Firefox'] } },
    { name: 'webkit', use: { ...devices['Desktop Safari'] } },
    { name: 'mobile-chrome', use: { ...devices['Pixel 5'] } },
    { name: 'mobile-safari', use: { ...devices['iPhone 13'] } },
  ],
});
```

## ✅ Вопросы для самопроверки

- [ ] Какие виды нагрузочного тестирования вы знаете?

<details>
<summary>Ответ</summary>

(1) **Нагрузочное** — поведение при ожидаемой нагрузке. (2) **Стресс-тестирование** — нахождение предела прочности. (3) **Тестирование всплесков** — реакция на резкое увеличение нагрузки. (4) **Тестирование выносливости** — стабильность при длительной нагрузке. Ключевые метрики: время отклика P95, пропускная способность, частота ошибок, использование CPU/памяти.

</details>

- [ ] Какие проверки безопасности должен выполнять QA?

<details>
<summary>Ответ</summary>

Базовые проверки по OWASP Top 10: (1) **Контроль доступа** — нельзя получить чужие данные. (2) **Инъекции** — SQL, XSS в полях ввода. (3) **Аутентификация** — блокировка при переборе. (4) **Конфигурация** — нет стектрейсов, стандартных паролей. (5) **Заголовки безопасности** — HTTPS, CSP, X-Frame-Options. QA не заменяет пентест, но ловит базовые уязвимости.

</details>

- [ ] Как организовать кросс-браузерное тестирование?

<details>
<summary>Ответ</summary>

(1) Определить матрицу поддержки (какие браузеры/ОС). (2) Приоритизировать по аналитике (какие браузеры используют пользователи). (3) Автоматизировать с Playwright (поддерживает Chromium, Firefox, WebKit). (4) Типичные проблемы: CSS-различия, рендеринг шрифтов, API браузера. (5) Тестировать критический путь во всех браузерах, остальное — в основном.

</details>
