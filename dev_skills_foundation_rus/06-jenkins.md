## 1. Что такое Jenkins?

**Jenkins** — это сервер автоматизации с открытым исходным кодом, используемый для CI/CD. Это самый распространённый инструмент CI/CD в корпоративных средах.

### Ключевые характеристики:
- **Локальная установка** — работает на серверах вашей компании
- **Основан на плагинах** — тысячи плагинов расширяют функциональность
- **Пайплайн как код** — пайплайны определяются в `Jenkinsfile`
- **Распределённый** — может запускать сборки на нескольких машинах (агентах)

## 2. Архитектура Jenkins

```
┌─────────────────────────────────────────────────────────────┐
│                     МАСТЕР JENKINS                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────────┐ │
│  │ Веб-     │  │Планиров- │  │ Очередь  │  │ Менеджер    │ │
│  │интерфейс │  │щик задач │  │ сборок   │  │ плагинов    │ │
│  └──────────┘  └──────────┘  └──────────┘  └─────────────┘ │
└───────────┬──────────────────────┬──────────────────────────┘
            │                      │
     ┌──────┴──────┐        ┌─────┴───────┐
     │  АГЕНТ 1    │        │  АГЕНТ 2     │
     │  (Linux)    │        │  (Windows)   │
     │  Выполняет  │        │  Выполняет   │
     │  сборки     │        │  сборки      │
     └─────────────┘        └──────────────┘
```

| Компонент | Роль |
|-----------|------|
| **Мастер** | Управляет всем: планирование, UI, конфигурация |
| **Агент (Нода)** | Машина, которая выполняет сборки |
| **Исполнитель (Executor)** | Слот на агенте, который может выполнять одну сборку |
| **Задача/Проект (Job)** | Определённая задача (сборка, тест, деплой) |
| **Сборка (Build)** | Одно выполнение задачи |
| **Пайплайн** | Серия стадий, определённых в коде |

## 3. Навигация по интерфейсу Jenkins

### Панель управления (Главная страница):

```
┌─────────────────────────────────────────────────────────────┐
│ Jenkins                                    [user] [выход]    │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  📊 Очередь сборок    │  Имя задачи     Последняя   Статус  │
│  (пусто)              │  ──────────────────────────────────  │
│                        │  my-app-pipeline   #142    ✅ Успех  │
│  🖥️ Исполнители       │  api-tests         #89     ❌ Провал │
│  Агент-1: свободен    │  nightly-build     #56     ✅ Успех  │
│  Агент-2: собирает    │  deploy-staging    #33     ⚡ Идёт   │
│                        │                                      │
└─────────────────────────────────────────────────────────────┘
```

### Иконки статуса сборки:

| Иконка | Значение |
|--------|----------|
| ☀️ Синий/Зелёный круг | Последняя сборка успешна |
| ⛈️ Красный круг | Последняя сборка упала |
| 🌤️ Жёлтый круг | Последняя сборка нестабильна (тесты упали) |
| ⚫ Серый круг | Никогда не собиралось / отключено |
| ⚡ Мигающий | Сборка идёт прямо сейчас |

### Ключевые страницы:

| Страница | Как попасть | Что показывает |
|----------|------------|----------------|
| **Панель управления** | Клик на логотип «Jenkins» | Обзор всех задач |
| **Страница задачи** | Клик на имя задачи | История сборок, конфигурация |
| **Страница сборки** | Клик на номер сборки (#142) | Детали сборки, логи |
| **Вывод консоли** | Клик «Console Output» на странице сборки | Полный лог сборки |
| **Результаты тестов** | Клик «Test Result» на странице сборки | Детали прохождения/падения тестов |
| **Вид пайплайна** | Клик «Pipeline» на странице задачи | Визуальный обзор стадий |

## 4. Jenkinsfile (Пайплайн как код)

Современный Jenkins использует **Декларативные пайплайны**, определённые в `Jenkinsfile`, который хранится в Git-репозитории проекта.

### Базовый Jenkinsfile:

```groovy
pipeline {
    agent any                          // Запуск на любом доступном агенте

    environment {
        APP_NAME = 'my-application'
        JAVA_VERSION = '17'
    }

    stages {
        stage('Build') {
            steps {
                echo 'Сборка приложения...'
                sh 'mvn clean compile'
            }
        }

        stage('Test') {
            parallel {                 // Запуск тестов параллельно
                stage('Unit Tests') {
                    steps {
                        sh 'mvn test'
                    }
                }
                stage('Integration Tests') {
                    steps {
                        sh 'mvn verify -P integration-tests'
                    }
                }
            }
        }

        stage('Package') {
            steps {
                sh 'mvn package -DskipTests'
                archiveArtifacts artifacts: 'target/*.jar'
            }
        }

        stage('Deploy to Staging') {
            when {
                branch 'develop'       // Только для ветки develop
            }
            steps {
                echo 'Деплой в staging...'
                sh './deploy.sh staging'
            }
        }

        stage('Deploy to Production') {
            when {
                branch 'main'
            }
            input {                    // Ручное одобрение
                message 'Задеплоить в продакшен?'
                ok 'Да, деплоить!'
            }
            steps {
                echo 'Деплой в продакшен...'
                sh './deploy.sh production'
            }
        }
    }

    post {
        always {
            junit 'target/surefire-reports/*.xml'    // Публикация результатов тестов
            cleanWs()                                 // Очистка рабочего пространства
        }
        success {
            echo '✅ Пайплайн прошёл успешно!'
        }
        failure {
            echo '❌ Пайплайн упал!'
            // Отправка уведомления (email, Slack и т.д.)
        }
    }
}
```

### Ключевые секции Jenkinsfile:

| Секция | Назначение |
|--------|-----------|
| `pipeline` | Корневой элемент — оборачивает всё |
| `agent` | Где запускать (на какой машине) |
| `environment` | Определение переменных окружения |
| `stages` | Список стадий пайплайна |
| `stage` | Именованная группа шагов |
| `steps` | Фактические команды для выполнения |
| `when` | Условия для запуска стадии |
| `input` | Ожидание ручного одобрения |
| `parallel` | Запуск стадий параллельно |
| `post` | Действия после завершения пайплайна |

### Типичные условия `when`:

```groovy
when { branch 'main' }                    // Только для ветки main
when { branch 'feature/*' }               // Любая feature-ветка
when { expression { env.DEPLOY == 'true' } } // Пользовательское условие
when { changeset '**/*.java' }             // Только если изменились Java-файлы
when { tag 'v*' }                          // Только для тегов версий
```

## 5. Чтение логов сборки Jenkins

**Console Output** (Вывод консоли) — это место, где вы найдёте всё, что произошло во время сборки.

### Пример: Лог успешной сборки

```
Started by user admin
Running on agent-1 in /var/jenkins/workspace/my-app
[Pipeline] Start of Pipeline
[Pipeline] stage (Build)
[Pipeline] sh
+ mvn clean compile
[INFO] Scanning for projects...
[INFO] Building my-application 1.0.0
[INFO] --- maven-compiler-plugin:3.11.0:compile ---
[INFO] Compiling 42 source files to /target/classes
[INFO] BUILD SUCCESS
[Pipeline] stage (Test)
[Pipeline] sh
+ mvn test
[INFO] Running com.example.UserServiceTest
[INFO] Tests run: 15, Failures: 0, Errors: 0, Skipped: 0
[INFO] BUILD SUCCESS
[Pipeline] stage (Package)
[Pipeline] sh
+ mvn package -DskipTests
[INFO] Building jar: /target/my-application-1.0.0.jar
[INFO] BUILD SUCCESS
[Pipeline] archiveArtifacts
Archiving artifacts
[Pipeline] End of Pipeline
Finished: SUCCESS
```

### Пример: Лог упавшей сборки

```
[Pipeline] stage (Test)
[Pipeline] sh
+ mvn test
[INFO] Running com.example.UserServiceTest
[ERROR] Tests run: 15, Failures: 2, Errors: 0, Skipped: 0
[ERROR] UserServiceTest.testCreateUser:45 Expected: 201, Got: 500
[ERROR] UserServiceTest.testDeleteUser:78 NullPointerException
[INFO] BUILD FAILURE
[Pipeline] }
[Pipeline] // stage
[Pipeline] stage (Declarative: Post Actions)
[Pipeline] junit
Recording test results
Test Summary: 13 passed, 2 failed
[Pipeline] End of Pipeline
Finished: FAILURE
```

### Как находить ошибки в логах:

1. **Прокрутите в конец** — финальный статус находится в конце
2. **Поищите `ERROR`** — найдите сообщения об ошибках
3. **Поищите `FAILURE`** — найдите упавшую стадию
4. **Ищите имена тестов** — определите, какие тесты упали
5. **Проверьте стадию** — какая стадия упала (Сборка? Тесты? Деплой?)

## 6. Jenkins для QA

### Что QA должен делать в Jenkins:

| Задача | Как |
|--------|-----|
| **Проверить статус сборки** | Смотреть на панель управления — сборка зелёная? |
| **Читать отчёты о тестах** | Клик на сборку → Результаты тестов |
| **Запустить сборку** | Клик «Build Now» или «Build with Parameters» |
| **Проверить статус деплоя** | Смотреть стадию деплоя — она прошла успешно? |
| **Найти детали упавших тестов** | Console Output → поиск FAILURE |
| **Получить артефакты сборки** | Клик на сборку → найти JAR/отчёты |

### Параметры сборки (параметризованные сборки):

Некоторые задачи Jenkins принимают параметры, которые можно задать перед запуском:

```groovy
parameters {
    string(name: 'ENVIRONMENT', defaultValue: 'staging', description: 'Целевое окружение')
    choice(name: 'BROWSER', choices: ['chrome', 'firefox'], description: 'Браузер для тестов')
    booleanParam(name: 'RUN_SMOKE_TESTS', defaultValue: true, description: 'Запустить smoke-тесты?')
}
```

Когда вы нажимаете **«Build with Parameters»**, вы можете задать эти значения перед началом сборки.
